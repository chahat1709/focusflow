🧑‍⚖️ External Architecture Review

Gemini 2.5 Pro — System Review
✅ Masterstroke

Decoupled Async Transport (L2 & L3)

Using asyncio.Queue to decouple Bleak GATT from DSP is textbook best practice. Bluetooth LE is notoriously finicky — if DSP blocks the BLE read loop, packets drop. 3-second ring buffers ensure processing spikes don't cause data loss.

✅ Masterstroke

Numba JIT Kernel (Layer 4)

Achieving 20µs per chunk in Python for an Adaptive NLMS filter is stellar. Python's GIL chokes on real-time sample-by-sample DSP, so dropping to LLVM machine code via @njit is the exact right move. The warm-start weight persistence shows deep understanding of adaptive filter convergence.

✅ Masterstroke

Offline-First Resilience (Layer 8)

Pushing data to local SQLite before syncing to Supabase ensures session data is never lost if internet drops — critical for uninterrupted focus sessions. A surprisingly rare design decision in BCI apps of this type.

✅ Masterstroke

Personalised Metrics (Layer 6 & 7)

Using Klimesch Individual Alpha Frequency (IAF) instead of static band ranges makes cognitive metrics significantly more accurate per user. Relying on TBR is a scientifically backed biomarker for focus and ADHD (Arns et al. 2013).

✅ Implemented — V2.2

Layer 9: 500ms HTTP Polling → SSE Push

The UI polled at 500ms (2Hz). Polling creates unnecessary HTTP overhead (headers, TCP handshakes) and caps UI responsiveness. Reviewer recommended WebSockets or Server-Sent Events.

✅ Upgraded to SSE 20Hz push stream.
Route: GET /api/stream · EventSource on frontend.
No TCP handshake per frame. Three.js sphere now fluid.
✅ Implemented — V2.2

Layer 5: 4-Channel CAR Blink Smear Risk

CAR on 4 channels risks smearing blink voltage from AF7/AF8 into TP9/TP10 before feature extraction. Code already used TP9/TP10-only reference, but still applied even during blink transients.

✅ Blink-Aware CAR: if frontal spike > 6× temporal std → skip CAR entirely.
_skip_car = True · temporal channels now fully protected.
✅ Implemented — V2.3

CFC Window Too Short (3s → 10s)

Only 12–24 theta cycles in a 3-second window — too few for stable Phase-Amplitude Coupling. AMI would jump erratically.

✅ Dedicated _cfc_buffer = deque(maxlen=2560) (10 seconds).
CFC now computes PAC over 40–80 theta cycles. Statistically reliable.
✅ Implemented — V2.3

FFT Resolution Trap (1Hz → 0.25Hz IAF bins)

nperseg=256 → 1Hz bins. IAF could only be 9.0 or 10.0 Hz, losing the precision that makes it a personalized metric.

✅ Added nfft=1024 to Welch call → 0.25Hz bins.
IAF now resolves 9.75Hz, 10.25Hz, etc. No latency impact.
✅ Implemented — V2.3

Muscle-Beta "Fake Focus" Confound

Jaw clenching injects noise into 20–30Hz (Beta band), falsely lowering TBR and triggering a "hyper-focused" reading.

✅ EMG-Beta cross-correlation: if 20–30Hz spikes WITH 40–100Hz simultaneously → tbr = 0.0. Focus metric frozen. No fake focus from jaw tension.
✅ Implemented — V2.3

NLMS Baseline Shift Death Lock

Headband shift → derivative spikes → gate freezes → filter permanently locked. Session data corrupted silently.

✅ frozen_count returned by kernel. If frozen > 640 samples (2.5s) → final_weights = zeros. Filter recovers immediately.
✅ Implemented — V2.3

Headband Impedance Drift Over 30+ Minutes

Sweat changes electrode impedance — raw RMS headband detection calibrated at minute 1 triggers false "off-head" at minute 40.

✅ Powerline antenna ratio check: If 50/60Hz energy > 60% of total signal → electrode is off-head (antenna mode). Sweat-immune.
✅ Implemented — V2.3

Hardcoded Port 5077 Collision

Antivirus or zombie process holding port 5077 → server silently fails to bind → blank screen for end user.

✅ Dynamic port binding: tries 5077 first, falls back to OS port 0. Writes %TEMP%/.focusflow_port. EXE never silently fails.
⏳ V3.0 Roadmap

Layer 10: 280MB PyInstaller Payload

PyInstaller unpacks ~280MB to a temp directory on cold start (5–15s). Can trigger Windows Defender false positives. Reviewer suggested Rust DSP layers via pybind11 + Tauri frontend for <50MB and instant startup.

Current approach is acceptable for client delivery. Rust/Tauri rewrite is the long-term V3.0 architecture goal.
